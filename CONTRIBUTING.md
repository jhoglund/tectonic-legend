# Git Workflow

## Branches

- **`main`** — always a working build. Web preview always loads, puzzles always solvable.
- **`feature/<name>`** — new features and experiments (`feature/progression-mvp`, `feature/daily-puzzle`).

## Day-to-day process

1. **Start new work:** branch from main.
   ```bash
   git checkout main
   git pull
   git checkout -b feature/<name>
   ```

2. **Commit regularly** — after each logical chunk:
   - New component works in the browser
   - Bug fix verified
   - Refactor complete
   - Open Design iteration accepted

3. **Push after every commit** — backup + recovery.
   ```bash
   git push -u origin feature/<name>   # first push
   git push                             # subsequent pushes
   ```

4. **Merge to main** when tested:
   ```bash
   git fetch origin
   git checkout feature/<name>
   git rebase origin/main          # or: git merge origin/main
   git checkout main
   git merge feature/<name>        # fast-forward
   git push
   git branch -d feature/<name>
   git push origin --delete feature/<name>
   ```

## Commit messages

- Short, imperative first line: `Add naked-single mastery counter`
- Optional body for the *why*, not the *what*
- Co-authored-by line when pair-programming with Claude

## Build verification

Always verify before merging to main:

```bash
npm run lint
npm run build
```

Once Capacitor is scaffolded, also dry-run the iOS build before merging anything that touches the platform layer.

## Rules

- Never force-push to `main`
- Don't merge broken builds to `main`
- Keep feature branches short-lived — days, not weeks
- Simple `git merge` — no squash, no rebase-of-merge (keeps full history)
- Always fast-forward the feature branch to `origin/main` before merging into main (fetch first)

## When merging in Open Design output

Prototypes generated by Open Design land under `prototypes/<date-or-slug>/`. They are tracked in git (so iteration history is visible) but **never built or imported by the app**. If a prototype graduates into shipping UI, the relevant code moves into `src/` and the prototype stays as the design reference.

See [`specs/design-workflow.md`](specs/design-workflow.md) for the full design ↔ code loop.
