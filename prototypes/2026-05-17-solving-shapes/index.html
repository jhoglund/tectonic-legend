Solving screen — shape exploration

Ten design directions for the Solving screen, all on one page — exploring grid/cell size, corner radius, number-key and toolbar-button shape & size, a keypad-first layout, and two full-width board treatments. Each variant is shown at both a 5×5 and an 8×8 grid. The boards are illustrative, not solvable puzzles. Session: 2026-05-17.

01 · Baseline

The Solving screen as it ships today — the reference point every other variant is judged against. Standard cell size, near-square cells, 8px buttons, 44px keypad keys, toolbar-then-keypad order.

cells 52/34px · board r6 · buttons r8 · keys 44px r8 · layout standard

02 · Soft & rounded

Everything generously rounded — a 22px board, fully-pill toolbar buttons, big-radius keypad keys. Slightly larger cells. The friendliest, most approachable read.

cells 54/35px · board r22 · buttons rpill · keys 52px r18 · layout standard

03 · Sharp & precise

Minimal radius across the board, buttons, and keys — crisp, engineered, logic-puzzle-serious. The opposite pole from "soft".

cells 52/34px · board r2 · buttons r2 · keys 46px r2 · layout standard

04 · Board-forward

The board is the hero — large cells on both grids, with a deliberately compact toolbar and smaller keypad keys so the puzzle gets the room.

cells 64/42px · board r8 · buttons r8 · keys 40px r8 · layout standard

05 · Keypad-first layout

The number keys sit directly under the grid, with the Notes/Hint/Validate/Clear bar below them. Otherwise baseline sizing — this isolates the layout-order change.

cells 52/34px · board r6 · buttons r8 · keys 44px r8 · layout keypad-first

06 · Big circular keypad

Large, fully-circular number keys placed right under the grid for thumb reach, with a compact pill toolbar below. Keypad-first layout.

cells 50/33px · board r12 · buttons rpill · keys 58px rcircle · layout keypad-first

07 · Compact & dense

Smaller cells, smaller keys, a slim toolbar and tighter gaps — everything pulled in so the whole screen fits comfortably on a small phone without scrolling.

cells 44/30px · board r5 · buttons r6 · keys 38px r6 · layout standard

08 · Card-framed

The board, toolbar, and keypad each sit in their own elevated card with a generous radius — a structured, sectioned screen where each control group reads as a distinct surface.

cells 50/33px · board r16 · buttons r10 · keys 46px r10 · layout standard · framed

09 · Full-bleed grid

The grid grows edge-to-edge — the board spans the full width of the screen with no side margin, square corners, cells scaling to fill. Maximum board, zero frame.

board full-bleed · board r0 · buttons r8 · keys 44px r8 · layout standard

10 · Inset grid (~10px)

The grid spans the full width less a ~10px margin on each side — nearly edge-to-edge, but with a slim breathing gap so the board still reads as a contained surface.

board full-width −10px · board r8 · buttons r8 · keys 44px r8 · layout standard
