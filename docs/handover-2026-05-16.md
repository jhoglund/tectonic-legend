## 2. Decisions taken — please review

### Daily puzzle (item 14)
- **Weekday → difficulty:** Mon–Tue Easy, Wed–Thu Medium, Fri Hard, Sat
  Expert, **Sun Easy**. ADR-0010 named Sunday "curated tutorial-style";
  with no curation pipeline, v1 simplifies Sunday to Easy.
- **All daily puzzles are 5×5.** Predictable; 8×8 dailies can come later.
- The generator gained a seeded PRNG (`mulberry32`); a seed makes a run
  fully deterministic. The streak counter is the existing profile
  streak (consecutive days with *any* solve), not daily-specific —
  simplest honest v1.

### Share artifact (item 15)
- **v1 is a text block, not a rendered image.** ADR-0004 said
  "canvas / SVG-to-PNG". Text (emoji grid, Wordle-style) is universally
  pasteable, needs no rendering pipeline, and is the mechanic that
  actually made Wordle viral. An image artifact can follow. **This
  deviates from ADR-0004 — confirm or override.**
- Per-cell colour: ⬜ clue · 🟩 solved unaided · 🟨 used a hint.
  "Corrections → red" from ADR-0004 is omitted — corrections aren't
  tracked. Shares via the native iOS share sheet, clipboard on web.

### Capacitor (item 21)
- **Bundle ID `com.jhoglund.tectonic`** — a working placeholder. It is
  effectively **permanent** once an App Store Connect record exists.
  **Confirm it before you create that record.** It need not match the
  brand name.
- `appName` "Tectonic" — working title only.
- Capacitor 8, Swift-Package-Manager based (no CocoaPods).
- `vite.config` `base` is now conditional: GitHub Pages keeps the
  project sub-path; `npm run build:ios` uses `/` so the native WebView
  resolves assets. New scripts: `build:ios`, `sync:ios`.

### Settings (item 19)
- Trimmed to **How to Play + About** — the App Store audit flagged the
  old stub as a guideline-2.1 dead-end. Theme / sound / haptics are
  **dropped from v1**: there is no theme-switching, no sound, and no
  haptics in the app, so toggles for them would be fake. They return
  with their features.

### Paywall (item 18) & v1 monetization posture
- `src/components/Paywall.tsx` is a complete, ready-to-wire UI. It is
  **not mounted anywhere** and gates **no** feature — so there is no
  "premium feature with no purchase path" (no App Store 3.1.1 risk).
- **Recommendation: ship the soft-launch v1 with no live IAP.** Wiring
  StoreKit needs your Apple enrolment, App Store Connect products, and
  the re-derived free/premium split — all decisions, not code. A clean
  free v1 validates retention first; IAP follows. **Your call.**
